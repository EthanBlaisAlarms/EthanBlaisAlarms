EBA OS Crashes

EBA OS Alert!

This page interacts with EBA OS directly. However, it doesn't look like you're viewing this page on EBA OS.

When EBA OS encounters an error it cannot recover from, EBA OS will crash. Generally, when EBA OS crashes, it'll display an error code describing the error. Sometimes, additional information about the crash may be displayed. After EBA OS crashes, your system will reboot automatically and EBA OS will run Automatic Repair to diagnose the cause of the crash. A detailed description of various errors is available below.

MANUALLY_INITIATED

MANUALLY_INITIATED occurs when a user manually initiates a crash. Generally, this is done by pressing CTRL+END. Since this error is manually initiated, no further action is required.

MISSING_OS_EXECUTABLE

MISSING_OS_EXECUTABLE occurs when EBA OS attempts to load a system executable, but the executable is missing.

NO_BOOTFILE

NO_BOOTFILE occurs when EBA OS attempts to boot normally, but the boot file is missing. You'll need to repair EBA OS to fix this error.

SYSTEM_EXCEPTION

SYSTEM_EXCEPTION occurs when a critical system file is missing while EBA OS is running.

UNKNOWN_OPERATING_SYSTEM

UNKNOWN_OPERATING_SYSTEM occurs when EBA OS is unable to read the system information file. You may need to repair EBA OS to fix this error.

UNKNOWN_PARTITION

UNKNOWN_PARTITION occurs when the recovery environment attempts to perform a repair action, but an EBA OS partition was not found.

UNMOUNTABLE_BOOT_VOLUME

UNMOUNTABLE_BOOT_VOLUME occurs when a required partition cannot be mounted. The partition may be missing. Sometimes, rebooting your system can fix this error. However, you may need to repair your installation of EBA OS if that does not work.
